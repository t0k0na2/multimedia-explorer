# Windows App SDK (WinUI 3) プロジェクト 開発基本ルール

## 1. プロジェクト概要

- **フレームワーク**: Windows App SDK (WinUI 3)
- **使用言語**: C#, C++
- **プロジェクト特性**: C#（主にUI/アプリケーションロジック）とC++（高性能なマルチメディア処理/コアロジック）が連携するアーキテクチャ。

## 2. コーディング基本ルール (共通)

- **【必須】ソースコード内のコメントはすべて日本語で記述すること。**
- コードの可読性と保守性を重視し、各言語の標準フォーマットやMicrosoftの命名規則に従うこと。
- クラス名、メソッド名は PascalCase、変数は camelCase を基本とする。

## 3. C# 開発ルール

- 主にフロントエンド(UI)の制御やアプリケーションの全体進行を担当する。
- **アーキテクチャ**: MVVM (Model-View-ViewModel) パターンを使用し、ビジネスロジックとUIを適切に分離すること。
- **非同期処理**: `async`/`await` を積極的に使用し、マルチメディア処理等の重い処理でUIスレッドをブロックしないこと。

## 4. C++ 開発ルール

- パフォーマンスが重視される処理や、画像・動画などのネイティブ処理を担当する。
- **モダンC++**: モダンC++ (C++17/C++20 等) の機能を最大限に活用すること。
- **メモリ管理**: 生ポインタによる手動管理を避け、必ずスマートポインタ (`std::unique_ptr`, `std::shared_ptr`) を用いてリソースリークを防ぐこと。
- **WinRT**: COM コンポーネントやネイティブAPI操作には、C++/WinRT (`winrt::com_ptr` など) と最新の標準ライブラリを活用すること。

## 5. C# と C++ の連携（相互運用）

- インターフェース境界を明確化し、P/Invoke や Windows Runtime (WinRT) コンポーネントを状況に合わせて適切に選択すること。
- コールバックやデータマーシャリングの際には、メモリのコピーやオーバーヘッドが最小限になるよう設計すること。
- 例外処理を徹底し、C++からのネイティブ例外がC#側で適切に捕捉されるように設計すること。

## 6. UI/UX (XAML & WinUI 3) ルール

- **Fluent Design**: Microsoft の Fluent Design System に準拠し、Mica や Acrylic などのマテリアルやモダンなアニメーションを使用すること。
- **レスポンシブ**: 動的なウィンドウサイズ変更にも対応できる、柔軟な XAML レイアウト設計を行うこと。

## 7. AI エージェント（アシスタント）向け規則

- 英語で思考してもよいが、**ユーザーへの返答、Artifact (Implementation Planなど) の出力、ソースコードへのコメント付与はすべて日本語**で行うこと。
